# EIP-7702 rollout for LORE

This repo is now 7702-ready, but the feature is intentionally disabled by default.

## What was added

- `contracts/LineaOre7702Delegate.sol`
  - Narrow game-specific delegate executor for 7702 EOAs.
  - Supports:
    - `approveAndPlaceBatchSameAmount`
    - `placeBatchSameAmount`
    - `claimRewards`
    - `claimEpochsRebate`
    - `resolveEpoch`
    - `claimResolverRewards`
- `app/lib/eip7702.ts`
  - Client-side capability/config helpers.
  - Calldata builders for the delegate contract.
- `app/hooks/usePrivyWallet.ts`
  - `signEip7702Delegation()`
  - `sendTransaction7702()`
  - `eip7702` capability object

## Required env

Set both:

- `NEXT_PUBLIC_EIP7702_ENABLED=1`
- `NEXT_PUBLIC_EIP7702_DELEGATE_ADDRESS=0x...`

Current Linea Sepolia delegate:

- address: `0x170067a88e64bba842ae6615ab277493de32629a`
- deploy block: `28203466`

Optional server mirrors:

- `EIP7702_ENABLED=1`
- `EIP7702_DELEGATE_ADDRESS=0x...`

## Intended rollout order

1. Deploy `LineaOre7702Delegate.sol` to Linea Sepolia.
2. Enable the env flags on Sepolia only.
3. Use the delegated `approve + batch bet` path in auto-miner and silent embedded-wallet bets.
4. Integrate reward and rebate claim flows.
5. Only after Sepolia validation, promote to mainnet.

## Safety notes

- The delegate contract is intentionally narrow and does not expose arbitrary target execution.
- Current implementation is best suited for self-sent delegated transactions and game batching.
- Sponsored gas / relayer flows should be added only after Sepolia validation.
- Do not allow arbitrary delegate addresses from the client. Keep the delegate address app-configured.
